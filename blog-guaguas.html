Nombre Guaguas: Exploring a Century of Chilean Baby Name Trends

What names were popular in Chile in the 1950s? Is "Mateo" really taking over? When did "José" peak? A fully serverless dashboard with an AI assistant to explore it all.

What names were popular in Chile in the 1950s? Is "Mateo" really taking over? When did "José" peak? These are the kinds of questions that Nombre Guaguas answers — a fully serverless dashboard that visualizes baby name trends in Chile from 1920 to 2025.

The word guagua comes from the Quechua word wawa and is used in Chile and other South American countries to refer to a baby. It felt like the perfect name for a project dedicated to exploring how Chileans have named their children over the past century.

The Data

The updated dataset until 2025 comes from Chile's Civil Registry (Servicio de Registro Civil e Identificación) and was originally compiled by Riva Quiroga for the years 1920-2021. It contains over 887,000 records spanning from 1920 to 2025, where each record represents a unique combination of name, gender, and year along with the number of registrations.

Before loading the data, every name goes through a normalization process: converted to lowercase, accents removed from vowels (so "María" becomes "maria"), but preserving the ñ (so "Muñoz" stays "muñoz"). This ensures that searches work consistently regardless of how users type a name.

The data lives in a Cloudflare D1 database — a serverless SQLite-compatible database that sits at the edge, close to users. With indexes on name, gender, year, and a composite index on all three, queries return results in milliseconds.

The Backend

The entire backend is a single Cloudflare Worker — a serverless function that runs at the edge across Cloudflare's global network. It exposes two main endpoints:

  • /api/names — Accepts a name, optional gender filter, and year range, then queries D1 and returns the time series data. Simple, fast, and stateless.
  • /api/chat — An AI-powered chat endpoint that lets users explore the data conversationally in both Spanish and English.

Rate limiting is handled through Cloudflare KV, capping requests at 10 per minute per IP address to keep things fair without needing any external infrastructure.

The AI Chat Assistant

One of the more interesting features is the built-in chat assistant. Instead of just offering a search box, users can type natural language queries like "Show me María and José since 1960" or "Compare Sofía and Martina".

Under the hood, this uses Groq's API with function calling. The AI has access to three tools:

  • visualize_names — Replace the current chart with new names
  • add_name — Add a name to the existing chart
  • clear_chart — Clear the chart entirely

When a user sends a message, two things happen in parallel: the main AI model processes the request and decides which tools to call, while a separate moderation model checks that the message is actually about names (blocking off-topic requests like recipes or general knowledge questions). This dual-model approach keeps the assistant focused and responsive.

The chat also maintains conversation history (last 10 messages) so users can have natural back-and-forth interactions, and it responds in the same language the user is browsing in.

The Frontend

The frontend is a static site served through Cloudflare Pages — no framework, no build step, just HTML, CSS, and JavaScript.

For charting, Plotly.js renders interactive scatter plots with spline interpolation, showing how names rise and fall in popularity over the decades. Users can hover over any point to see exact counts, and export charts as high-resolution PNGs.

Bootstrap 5 provides the responsive layout, while noUiSlider powers the year range selector, letting users zoom into specific decades with a smooth drag interaction.

The interface supports both Spanish and English through a lightweight i18n system. Spanish is the default (this is a Chilean project, after all), and users can switch to English via a URL parameter. The language preference carries through to the AI chat, chart labels, and all UI elements.

Name pills appear below the chart as colored tags that match the chart lines, and users can click the × on any pill to remove that name from the visualization. The URL updates in real time with the current selection, making it easy to share specific comparisons with a link.

The Infrastructure

The entire application runs on Cloudflare's edge network with zero traditional servers:

  • Workers for the API logic
  • D1 for the database
  • Pages for the static frontend
  • KV for rate limiting state

This means the app runs close to users worldwide, scales automatically, and costs practically nothing to operate. Deployment is a single wrangler deploy command for the backend, and the frontend auto-deploys on every git push.

Try It Out

The dashboard is live at guaguas.pages.dev. Search for your name, compare it against others, or just chat with the AI to explore trends you never knew existed in Chilean naming history.

Data source: guaguas by Riva Quiroga, based on data from Chile's Servicio de Registro Civil e Identificación.
